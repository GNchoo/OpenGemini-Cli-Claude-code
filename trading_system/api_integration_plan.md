# 업비트 API 연동 계획

## 1. 개요
트레이더 마크 📊 시스템을 업비트 실제 거래와 연동하기 위한 계획

## 2. 개발 단계

### Phase 1: 환경 설정 및 기본 연동 (1-2일)
- [ ] 업비트 개발자 계정 생성
- [ ] API 키 발급 (읽기/쓰기 권한)
- [ ] 테스트넷 접속 정보 확인
- [ ] Python 업비트 라이브러리 설치 (`pyupbit`)
- [ ] 기본 연결 테스트

### Phase 2: 데이터 연동 모듈 개발 (2-3일)
- [ ] 실시간 시세 데이터 수집 모듈
- [ ] 잔고 및 포지션 조회 모듈
- [ ] 기본 주문(매수/매도) 모듈
- [ ] 주문 상태 확인 모듈
- [ ] 예외 처리 및 에러 핸들링

### Phase 3: 자동 트레이딩 엔진 개발 (3-4일)
- [ ] 신호 → 주문 변환 로직
- [ ] 리스크 관리 통합
- [ ] 포지션 사이즈 계산기
- [ ] 주문 실행 스케줄러
- [ ] 실시간 모니터링

### Phase 4: 모니터링 및 리포트 시스템 (2-3일)
- [ ] 실시간 대시보드
- [ ] Telegram 알림 시스템
- [ ] 일일/주간 리포트 자동화
- [ ] 성과 분석 도구
- [ ] 로깅 및 감사 추적

## 3. 기술 스택

### 필수 라이브러리
```python
# 업비트 API
pyupbit

# 데이터 처리
pandas, numpy

# 웹소켓 (실시간 데이터)
websocket-client

# 스케줄링
schedule, apscheduler

# 로깅
logging

# 환경 변수
python-dotenv
```

### 디렉토리 구조
```
trading_system/
├── api/
│   ├── __init__.py
│   ├── upbit_client.py      # 업비트 API 클라이언트
│   ├── data_stream.py       # 실시간 데이터 스트림
│   ├── order_manager.py     # 주문 관리
│   └── account_manager.py   # 계정 관리
├── trading/
│   ├── auto_trader.py       # 자동 트레이딩 엔진
│   ├── risk_manager.py      # 리스크 관리
│   └── position_sizer.py    # 포지션 사이즈 계산
├── monitoring/
│   ├── dashboard.py         # 실시간 대시보드
│   ├── notifier.py          # 알림 시스템
│   └── reporter.py          # 리포트 생성
└── config/
    ├── api_config.json      # API 설정
    └── trading_config.json  # 트레이딩 설정
```

## 4. 보안 및 안전장치

### API 키 관리
- 환경 변수 사용 (.env 파일)
- 암호화 저장

### 거래 제한
- 일일 거래 한도: 자본의 20%

- 단일 주문 한도: 자본의 5%

## 10. 성공 기준

### 기술적 기준
- 99.9% API 가용성
- 1초 이내 주문 실행
- 실시간 데이터 지연 < 100ms

### 트레이딩 기준
- 양수 기대 수익률
- 샤프비율 > 1.0
- 최대 낙폭 < 20%

### 운영 기준
- 24/7 무중단 운영
- 자동 복구 기능
- 실시간 모니터링
